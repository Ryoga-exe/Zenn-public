## COJT とは？

[筑波大学の情報学群](https://inf.tsukuba.ac.jp/)で行われている「組み込み技術キャンパス OJT」という授業のことです。
主に情報学群の3年次を対象としていて、ハードウェアコースとソフトウェアコースの2つに分かれています。

講師や TA のサポートを受けながら、実際に手を動かして開発を進めていくのが特徴です。

https://www.cojt.or.jp/tkb/

その中でも私はハードウェアコースの 17 期生として受講しました。

### ハードウェアコースについて

ハードウェアコースでは、受講者それぞれに FPGA ボードが与えられ、それを使って「映像信号の入出力」をテーマに演習を行います。

大まかな流れとしては、次のような課題に取り組みます。

- パターン表示回路
- 表示回路
- フィルタ回路
- 描画回路
- 自由課題

#### パターン表示回路

最初の課題では、単純なパターンを画面に表示します。  
ここでは主に、VGA 信号の扱い方や、映像を表示するための基本的なタイミングについて学びます。

#### 表示回路

次の課題では、メモリ上にある画像データを読み出して表示します。  
AXI バスのマスターとしてメモリを読み込む必要があるため、AXI の read transaction、つまり AR チャネルと R チャネルが登場します。

#### フィルタ回路

フィルタ回路では、入力画像に対してフィルタ処理を行い、その結果を AXI 経由でメモリに書き戻します。  
その後、表示回路を使って処理後の画像を表示します。  
この課題では read だけでなく write も必要になるので、AW, W, B チャネルまで含めて AXI を扱うことになります。

#### 描画回路

描画回路では、レジスタ経由で描画コマンドを受け取り、それに応じてメモリ上に図形などを書き込む回路を実装します。  
これまでの課題で学んだ AXI によるメモリ操作や表示の仕組みを組み合わせる、いわば総合演習のような課題です。

コマンド処理も含めて回路の構成がかなり複雑になるので、非常に実装が大変でした。

#### 自由課題

最後は、それまでの課題で学んだ内容を踏まえて自由に作品を作る課題です。  
自分はここで CPU を作り、その上で動く OS まで実装しました。  
もちろん、自作した CPU から描画コマンドを叩けるようにして、それまでに作ってきた描画回路も活用しています。
